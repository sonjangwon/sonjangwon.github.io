Project 03

Exchange
Migration Advisor

온프레미스 Exchange 환경을 자동으로 분석해 Microsoft 365 마이그레이션 전략을 진단·추천하는 시스템. 환경 정보를 입력하면 마이그레이션 방식(Cutover / Staged / Hybrid)을 자동 판단하고, 단계별 전략과 주요 체크리스트를 즉시 제공합니다.

문제 정의 & 해결
Before
  • Exchange → M365 마이그레이션 방식은 환경에 따라 크게 달라지지만 직접 판단하기 어려움
  • 담당자가 Exchange 버전, 사서함 수, 하이브리드 구성 여부 등을 수동으로 파악하고 전략 수립
  • Microsoft 공식 문서와 체크리스트를 일일이 확인해야 해 시간 소요가 큼
  • 조직마다 환경이 달라 일반적인 가이드가 그대로 적용되지 않는 경우 多
After
  • 환경 정보를 입력하면 Cutover / Staged / Hybrid 방식을 자동으로 판단
  • Exchange 버전·사서함 수·디렉터리 동기화 여부 등 핵심 변수 기반으로 진단
  • 단계별 마이그레이션 전략과 주의사항을 즉시 리포트로 제공
  • Copilot 에이전트를 통해 자연어로 질의하고 맞춤형 전략 추천 수령 가능

직접 설계 & 구현한 것
  • 01
    환경 진단 로직 설계 진단 엔진
    Exchange 버전, 사서함 수, 디렉터리 동기화 여부, 하이브리드 구성 유무 등 핵심 변수를 기반으로 Cutover / Staged / Hybrid 마이그레이션 방식을 자동 판단하는 로직 설계. 각 방식별 적합 조건과 제약 사항을 반영해 오진을 최소화.
    Cutover Staged Hybrid 조건 분기 로직
  • 02
    Copilot Studio 에이전트 구현 Copilot Studio · Power Automate
    사용자가 자연어로 환경 정보를 제공하면 단계적으로 필요한 정보를 수집하는 대화 흐름 설계. 수집된 정보를 진단 엔진에 전달해 마이그레이션 전략과 체크리스트를 자동 생성. Power Automate 연동으로 결과를 메일로 발송하는 기능 포함.
    대화 흐름 Power Automate 메일 발송
  • 03
    마이그레이션 전략 리포트 생성 리포트 자동화
    진단 결과에 따라 단계별 마이그레이션 절차, 예상 기간, 사전 준비 사항, 주의사항을 포함한 맞춤형 리포트를 자동 생성. Exchange 버전별 지원 종료 일정 및 전제 조건도 함께 제공해 의사결정에 필요한 정보를 한 번에 전달.
    단계별 절차 체크리스트 EOS 일정
