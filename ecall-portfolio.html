문제 정의
Problem
  • 연간 112 약 1,900만 건·119 약 1,200만 건 — 신고량 급증
  • 통화와 수기 입력 동시 처리로 인지 과부하 발생
  • 다양한 신고 유형에 매뉴얼을 즉시 적용하기 어려운 구조
  • 2022년 이태원 사고 — 초동 대응 미숙 반복
Solution
  • 실시간 STT + 화자 분리로 수기 입력 부담 제거
  • GPT-4o-mini로 위험도·유형·위치 자동 분석
  • 사건 유형별 체크리스트·매뉴얼 자동 추천
  • AI 추천 출동 부대 자동 배차 지원

담당 역할
  • 01
    위험도 5단계 산출 로직 핵심 담당

    GPT-4o-mini의 문맥 분석(AI Analysis)과 키워드 패턴(Keyword Patterns)을 통합하는 하이브리드 방식으로 설계했습니다. 실제 112·119 운영 기준을 분석하여 위험도 분류 체계를 직접 정의했으며, 단일 신호 대비 고위험 신고의 오분류율을 낮췄습니다.

    GPT-4o-mini Keyword Pattern Matching 위험도 1–5단계
  • 02
    RAG 기반 도메인 지식 베이스 핵심 담당

    긴급신고 도메인에 특화된 용어·표현 지식 베이스를 구축하고 RAG 구조로 STT 결과를 문맥 보정했습니다. 신고 유형별 핵심 용어(심정지, 압사, 교통사고 등)의 인식 오류를 줄이고 AI 분류 입력 품질을 높였습니다.

    RAG 도메인 지식 베이스 문맥 보정
  • 03
    사건 유형 분류 · 출동 부대 추천 핵심 담당

    재난/재해, 의료/응급, 범죄, 교통사고, 구조/구급, 기타 6개 유형 분류 체계를 구현했습니다. 분류 결과와 위험도, 팀 구성원 수·평균 경력을 종합하여 최적 출동 부대를 자동 추천하는 로직을 설계했습니다.

    6유형 분류 출동 부대 추천 GPT-4o-mini
  • 04
    AI 성능 벤치마크 설계 공동 참여

    GPT-3.5-turbo·GPT-4o·GPT-4o-mini 3개 모델을 50개 시나리오 × 3회 반복(150회)으로 실측 비교했습니다. 정확도 40%·응답속도 25%·비용 25%·일관성 10% 가중치를 직접 정의하여 GPT-4o-mini를 최종 선정했습니다.

    Python 벤치마크 스크립트 50개 시나리오 모델 비교 평가
  • 05
    실시간 대시보드 시각화 공동 참여

    신고 현황·위험도 분포·건수 통계를 실시간으로 표시하는 대시보드 개발에 참여했습니다. Dispatch Console에서 출동 부대의 이동 동선을 실시간으로 모니터링할 수 있습니다.
