문제 정의 & 해결
Before
  • M365 제품 업데이트는 시기와 범위를 예측하기 어려워 지속적인 추적 필요
  • 담당자가 직접 M365 Roadmap 페이지를 주기적으로 방문해 수동 확인
  • 제품별 업데이트 추적에 반복 공수 발생
  • 로드맵 변경 인지 후 팀 공유까지 별도 작업 필요
After
  • Power Automate 스케줄로 매일 자동 데이터 수집 및 갱신
  • Azure Blob Storage 캐싱으로 빠른 응답, 중복 API 호출 제거
  • Copilot 에이전트에 자연어로 질의해 즉시 확인 가능
  • 신규/변경 아이템 감지 후 Exchange Online으로 자동 메일 발송
시스템 아키텍처
① 자동 배치 플로우 — 매일 오전 3시
② 온디맨드 플로우 — 사용자 질의
직접 설계 & 구현한 것
  • 01
    M365 Roadmap API 수집 모듈 Azure Functions · Python
    데이터 수집·배치 실행·상태 확인 기능을 담당하는 각각의 함수를 Python으로 작성. 요청이 들어오면 해당 제품의 최신 로드맵 데이터를 가져와 저장 가능한 형태로 변환. Azure에 배포 및 운영 환경 구성까지 직접 진행.
  • 02
    로드맵 데이터 캐싱 — Azure Blob Storage Azure Blob Storage
    수집한 로드맵 데이터를 Azure Blob Storage에 저장해 재사용. 24시간 이내 같은 데이터 요청이 오면 외부 API를 다시 호출하지 않고 저장된 데이터를 바로 반환. 최대 108개(36제품×3상태) 파일을 관리하며 응답 속도 개선 및 비용 절감 효과.
  • 03
    Copilot Studio 에이전트 설계 & 배포 Copilot Studio · Microsoft Teams
    사용자가 제품과 상태를 선택하면 Azure Function을 호출해 결과를 반환하는 대화 흐름을 설계. REST API 연결 및 Power Automate 흐름 연동. Microsoft Teams에 에이전트 게시 완료.
  • 04
    Power Automate 일일 배치 오케스트레이션 Power Automate · Exchange Online
    매일 오전 3시 자동으로 로드맵 데이터를 수집·갱신하는 배치 플로우와, 결과를 이메일로 발송하는 플로우 구성. Function 실행 성공 시에만 메일이 발송되도록 조건 처리. 30일간 성공률 100%, 오류 0건.
